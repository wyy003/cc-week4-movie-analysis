## 学到的概念

### 数据分析
- DataFrame / Series / groupby 的业务含义
- 数据汇总和分组统计
- 数据质量检查的重要性
- 数据污染的影响

### 可视化
- 柱状图：适合展示排名和对比
- 饼图：适合展示占比关系
- 折线图：适合展示时间趋势
- 图表需要清晰的标题、标签和数值

### 业务思维
- 先探索多种方案，再选择最合适的
- 每个分析都要有明确的业务问题
- 报告要让非技术人员也能看懂
- 数据验证比代码正确性更重要

## 协作方式的进步

### 与 CC 的协作
- 用业务语言提需求，不用技术术语
- 明确验收标准（"我要看到什么"）
- 通过输出验证，不读代码
- 发现问题时让 CC 加日志输出

### 验证方法
- 打开图片查看效果
- 用计算器手动验证数字
- 对比预期答案
- 故意污染数据测试健壮性

## 最有效的 3 个 Prompt

1. **数据质量检查**
   ```
   帮我检查数据质量，包括：NaN 值、重复数据、
   日期格式、异常值、分布情况
   ```

2. **图表生成**
   ```
   画一张柱状图：X 轴是电影名，Y 轴是票房，
   标题"全球票房 Top 10"，保存为 chart_top10.png
   ```

3. **业务方案探索**
   ```
   我想看年度票房趋势，但不确定是看总票房还是平均票房。
   给我两种方案的对比，帮我决定用哪个。
   ```

## 遇到的挑战

1. **业务决策**：重复电影名（狮子王）是否合并？
   - 解决：分开算，因为是不同版本

2. **方案选择**：年度趋势用总票房还是平均票房？
   - 解决：用平均票房，消除电影数量影响

3. **数据验证**：如何确保数字准确？
   - 解决：C3 验证 + 手动计算 + 数据污染测试
